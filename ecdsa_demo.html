4.3   Signing a Bitcoin

This page is a live illustration of Algorithms 3.2, 3.3, 3.4 and 3.5 using the real secp256k1 domain parameters from Section 4.1. All field operations (point addition, point doubling and scalar multiplication) use Jacobian coordinates (Table 2.12, Appendix A.2.3). A single field inversion then converts the final Jacobian results back to affine.

Note that, in real life, this final inversion never happens, as it defeats the whole purpose of using projective coordinates.

Algorithm 3.5

Algorithm 3.5:
Line 1 — Verify r, s ∈ [1, n − 1]
Line 2 — Compute e = SHA-256(SHA-256(m))
